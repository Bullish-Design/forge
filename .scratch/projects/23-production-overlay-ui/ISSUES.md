## How to Use This File
When you hit a blocker that requires 3+ failed attempts to resolve, or that represents a significant architectural decision:
1. Create a new `ISSUE_<num>.md` file in this project directory
2. Document what was tried, what failed, and why
3. Link it here with the date and current status

Format:
```
## ISSUE_1: (Title)
- **Status**: open/blocked/resolved
- **Created**: YYYY-MM-DD
- **Last updated**: YYYY-MM-DD
- **Detailed log**: See [ISSUE_1.md](ISSUE_1.md)
```
