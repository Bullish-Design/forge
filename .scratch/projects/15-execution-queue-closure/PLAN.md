# PLAN

## Mandatory Rule
NO SUBAGENTS. All work is performed directly in this session.

## Objective
Create a new project tracker that converts outstanding work into a prioritized, actionable execution queue with milestones and completion criteria.

## Steps
1. Synthesize remaining work from existing project status artifacts.
2. Define priority tiers, owners, effort, dependencies, and sequencing.
3. Create a copy/paste-ready checklist suitable for active execution tracking.
4. Populate standard project template files with clear status, decisions, risks, and assumptions.

## Acceptance Criteria
- A new numbered project directory exists under `.scratch/projects/`.
- Standard files exist and are populated: `PROGRESS.md`, `CONTEXT.md`, `PLAN.md`, `DECISIONS.md`, `ASSUMPTIONS.md`, `ISSUES.md`.
- A clear execution queue with milestones and dependencies is documented.
- The checklist is immediately usable to track implementation completion.
